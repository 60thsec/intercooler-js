Intercooler 0.4.0 Changes

Change List

  • Added the X-IC-Trigger response header to trigger client side event handlers.
  • Added the ic-confirm attribute.
  • Merged a fix for an IE8 compatibility issue.

Commentary

The big change (small code change, but big conceptual change!) is the X-IC-Trigger header, which finally solves a tricky problem I've been wrestling with: How do you communicate server side state changes that have client-side UI ramifications that are outside the usual Intercooler request-and-replace flow?

This new mechanism allows you to cleanly separate your server side logic and your client-side logic even in cases where the normal flow isn't enough.

A great example is if you want to hide a modal if and only if a form in the modal saved properly via an intercooler request.
