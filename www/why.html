Why Use Intercooler?

The primary advantage of Intercooler is simplicity.

Using simple, declarative attributes you can build rich, interactive user interfaces in your application while avoiding the overhead (and sometimes, overkill) of MVC frameworks.

With intercooler.js, you don't need client-side models, routing, validation, rendering, factories, dependency injection, etc, eliminating a major source of complexity, inconsistency and security problems in your application.

How Does Intercooler Work?

Intercooler is a Partial View Controller framework: it communicates with the server via AJAX, but rather than using JSON responses, it uses fragments of HTML (i.e. Partial Views), which are then loaded into the DOM directly.

This is in contrast with older, more traditional MVC frameworks such as Ember.js or Angular.js, which use JSON and client-side rendering.

The AJAX response content is a partial bit of HTML, and it is swapped into the DOM directly.

Examples of some PVC frameworks are Turbolinks and pjax.

Intercooler formalizes this pattern using simple, declarative attributes, making PVC-style programing easy, flexible and fun.
