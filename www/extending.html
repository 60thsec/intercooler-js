Intercooler JS API

Intercooler has a few different extension points. You can call a few direct methods on the global Intercooler object, and you can also listen for intercooler related events.

API

Method Description
Intercooler.setLogger(logger) Sets a logger for intercooler to use (defaults to window.console
Intercooler.setLogLevel(Intercooler.logLevels.DEBUG) Sets the logging level for Intercooler
Intercooler.addURLHandler(handler) Lets you add a client side mock URL handler for testing/prototyping: 
  Intercooler.addURLHandler({
    'url': '/update',
    'get' : function() {
      return 'Here's some content!"
    },
  });
Intercooler.refresh(elt) Refreshes the given element manually

Events

Intercooler fires the following events on elements:

Event Description
ic.beforeHeaders(elt, xhr) Triggered before intercooler headers are processed.
ic.afterHeaders(elt, xhr) Triggered after intercooler headers are processed.
ic.beforeSend(elt, data) Triggered before sending an intercooler AJAX request to the server. The second argument to the event is the data hash, and can be added or removed from to change the values sent to the server.
ic.success(elt, data, textStatus, xhr) Triggered after a successful intercooler request is received
ic.error(elt, req, status, str) Triggered after an error occurs during an intercooler request
ic.complete(elt, data) Triggered after an intercooler request completes, regardless of status
ic.onPoll(elt) Triggered before a poll request is dispatched

Example

Here is some code that uses the BlockUI library to block the UI when an intercooler request is in flight:

        $(function(){
          $('.btn').on('ic.beforeSend', function(){
            $.blockUI();
          }).on('ic.complete', function(){
            $.unblockUI();
          });
        })
