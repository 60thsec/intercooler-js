intercooler.js

Add AJAX to your web application using simple, declarative HTML attributes.

Intercooler v0.4.5 has been released! Join our newsgroup to stay up to date. 
  <-- This button posts to /example/click when it is clicked -->
  <button ic-post-to="/example/click">
    Click Me!
  </button>

  <-- This span depends on the button above,
         and is refreshed after it is clicked -->
  <span ic-src="/example/click">
     You have not yet clicked the button
  </span>
  You have not yet clicked the button

Declarative

With Intercooler you can use easy to understand, declarative attributes to bind your HTML elements to AJAX end points.

Intercooler is a pragmatic library, however, so you can drop into javascript when it's necessary.

Incremental

You can use Intercooler for as much or as little of your application as you like.

No need to wholesale adopt a new framework to start adding AJAX functionality to your application!

Plays Well With Others

Intercooler is a little library built to play well with other web technologies, like AngularJS, EmberJS and Turbolinks.

Use the right tool for the particular job: Intercooler won't get in the way.

Dependency Detection

Intercooler uses pattern matching in your URLs to detect and refresh dependencies between HTML elements. Typically it will do the right thing without much work on your part. And, if you need to add or ignore dependencies, Intercooler allows you to do that as well.

Framework Agnostic

Although it was inspired by Partials in Rails, IntercoolerJS works equally well with any back end platform.

IntercoolerJS works great with any HTML/CSS framework, but was designed with Bootstrap in mind, making it easy to create dynamic UIs using the standard, simple Bootstrap techniques.

Open

IntercoolerJS was designed to be open and extensible so, when push comes to shove, you can customise pretty much any part of the it you need to.
